SCSS basics

SCSS is a very powerful extension of simple CSS. However, when SCSS code is executed, it will turn back into normal CSS code compiled.

In order to better visualize the process of conversion, the following SCSS logic can also be tested in this SCSS to CSS converter by pasting the SCSS code on the left side. The resulting CSS code then appears on the right side.

When creating larger projects, it is important to keep track of the to retain project-wide CSS logic, which often extends over several to added many files. Especially with several developers it grows the risk of writing redundant code or generally losing track of things.

In order to make the creation of CSS logic more compact and clear, a few functionalities of SCSS are presented in detail:

  1. Variables
  2. Nesting
  3. Imports
  4. Inheritance
  5. Mixins
  6. Conditions
  7. Functions

If your own IDE does not have a suitable precompiler, one must be installed to use SCSS. The Webseite of Sass or SCSS offers good instructions for this and a list of suitable tools if you prefer a UI.

As an alternative to SCSS, you can also write Sass code. The difference to SCSS is purely syntactic and, similar to Python, Sass would work with line indentations instead of curly braces and semicolons.

Further information on SCSS and Sass can be found on the website.

1. Variables in SCSS

With SCSS it is now possible to work with variables. This allows values ​​that are repeated within the CSS file (or even across files) to be summarized in one place. In this way, colors, sizes, distances, etc., which occur in the entire application, can be managed in a central position. This also saves the developer the time to search for all entries when adjusting the style of the website and only needs to change one place in the code instead.

In the example, the colors of the cubes are now saved in variables. Now the variables can be used in the code instead of the assigned color. It is also possible to create lists with which values ​​can be linked.

2. Nesting

Nesting CSS logic is a good option for avoiding redundant code, avoiding unnecessarily long class hierarchies, and keeping the code generally clean and tidy.

In the example, all cubes that are located in the "nesting container" class are given a white number by means of nesting. What also works, and is used at this point, is the use of media querys within this nesting.

In addition, it is possible to split class names using the "&" character in order to enable nesting. This is also required for pseudo classes like ":hover" for a mouseover effect.

3. Import

@Import has nothing to do with the classic CSS import-command. With @import the contents of SCSS files are integrated into other SCSS files. In this case the @import command is more like an @include.

SCSS files that are added to the main SCSS file via import are marked with a "_" underscore character at the beginning of the file name. These files are not created again when converting to CSS logic. Instead, their contents are integrated directly into the file(s), where they are integrated using @import. When importing the SCSS file, the underscore can be omitted.

In the example, the backgrounds for the cubes are imported from an external SCSS file "_scss-skills-extern.scss". Due to the preceding underscore in the file name, no CSS file with the same name is generated when converting to CSS logic. Only "scss-skills-extern" is used for the import command.

4. Inheritance / Extend

With @extend the logic of classes can be integrated into newly defined classes. If classes, which you include in other classes with @extend, are not used explicitly in the HTML code and only serve as a template, you can use "%" instead of "." be marked. When converting SCSS to CSS, content is only integrated where it has been included with "@extend".

In the example a new container "example-container-2" is created. This receives all properties from the class "example-container" inherited via the upper extend.

The background colors of the cubes are fetched from "% extend-cubes" via another extend. This class is never created itself when converting to CSS and is only used to inherit properties of classes, like "example-container-2" here.

5. Mixins

They are basically what functions are in other programming languages. You can create methods in one place in the code and use them multiple times. It is best practice to store these mixins in a separate folder "styles / mixins". Just like a function, a mixin can receive and process parameters.

In the example, the mixin "get-cube-properties" receives the number of the cube and supplies the required background. This is then added to the nested cube classes here using "@include". The explanation of the conditions used in the mixin are explained in the next section.

6. Conditions (if-else / for / while / each)

SCSS extends CSS by the possibility to use the following conditions:
  • @if() - @else if() - @else (see example)
    • With "@if" the subsequent condition is checked for its truth value
      • If true, the properties specified in the body are used
      • If wrong, any existing alternative conditions are checked
    • with "@else if" an alternative condition is checked for its truth value
    • with "@else" an alternative is created if the previous conditions did not apply
  • @for (see example)
    • "@for" - runs as a loop until the associated tag has reached a limit value
    • "$i" - Example of a variable, is automatically incremented after the loop has been run through
    • "from X through y" - the start value (X) and limit value (Y) are determined here
      • The difference between the start and limit value determines the number of loop runs
  • @each (see example)
    • can iterate over lists of different complexities, which can also consist of several values
      • one value - @each $social in facebook, twitter, instagram
      • two values - @each $header, $size in (h1: 32px, h2: 28px, h3: 24px, h4: 20px)
      • three values - @each $header, $size, $color in (h1, 32px, #333), (h2, 28px, #222) etc.
    • The number of runs depends on the length of the list
  • @while
    • the procedure is similar to the for-loop
    • receives a variable and a limit value
      • example: @while( $i <= 10 ) ...
    • incrementation of the variable must be entered
      • example: $i: $i + 1
  • Linking a loop variable with a class name
    • see class "example-6-cube-" in the for loop of the example
