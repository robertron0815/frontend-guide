HTML Coding Guideline

1. Valid and readable markup

  • write everything lowercase
  • take care of the correct indentation
    • spaces
  • 120 chars per line
    • settings.json => "editor.rulers": [ 120 ]
  • closing selfclosing-tags uniformly or not
    /
  • use automatic code formatting (only for new projects!)
    • settings.json => "editor.formatOnSave": false/true
  • organize DOM-Elements:
    • rather use semantic HTML5-elements instead of just a "div"
    • ask yourself if you need a block-element or not
      • (span {{"{display: block}"}} => just div)

2. Don‘t use inline styles and scripts

  • avoid inline styles
    • markup looks terrible, if it‘s used often
    • it‘s hard to override those styles by extern css
    • there are no excuses and exceptions {{";-)"}}

3. Script tags on the bottom (outside angular focus)

If you have an angular project, then load external scripts via angular.json

  • when the script file is downloaded, the browser is waiting for it and pauses the rendering
    1. Header: waiting for downloading and execution
    2. Body with async attribut: rendering is paused until the file is executed
    3. Body with defer attribut: download and execution following after the rendering

4. Performance

  • compress 3rd party (css/js) libraries
  • compress jpg and png images (e.g. https://tinypng.com/)
  • create different image sizes for different devices (jpg/png)
    • use srcsets and sizes

5. Validation

  • Use a validation tool in your IDE or in your browser
    • Chrome: Html Validator
    • Firefox: Html Validator
  • Validation plugin for VS Code
